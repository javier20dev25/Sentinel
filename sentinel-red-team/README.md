# Sentinel Red Team Suite
**Adversarial Security Benchmarking & Payload Generation**

This suite contains the internal tools used to "battle-test" the Sentinel Oracle Brain. The philosophy behind Sentinel is **Adversarial Training**: a security engine is only as strong as the attacks it has survived during development.

## Components

### 1. `payload_generator.js` (formerly SARB Mutator)
An automated metamorphic engine that generates uniquely mutated JavaScript attack vectors.
- **AST Mutations**: Wraps payloads in complex logic structures.
- **Encoding**: Uses Hex, Base64, and CharCode splitting.
- **Evasion**: Injects dead-code and invisible Unicode characters to break regex patterns.

### 2. `adversary_orchestrator.js` (formerly SARB Runner)
The orchestration engine that launches mass attacks against Sentinel.
- **Scale**: Can fire thousands of unique mutations in seconds.
- **Metrics**: Tracks Sentinel's FNR (False Negative Rate) against specific evasion tactics.

### 3. `corpus_manager.js`
A native laboratory manager that provisions real-world targets.
- **Git Integration**: Securely clones top-tier repositories (Webpack, Next.js, etc.).
- **NPM Provisioning**: Unpacks packages without executing lifecycle scripts to keep the host safe.

### 4. `firing_range_benchmark.js`
The systematic validation engine ("The Firing Range").
- **Multi-Profile**: Tests against `STRICT`, `BALANCED`, and `DEV` policies.
- **Performance**: Measures Latency, Memory Delta, and Global Risk Scores.

---

## The "Battle-Tested" Methodology
Sentinel was subjected to 1,700 unique mutations generated by this suite. Its current mathematical damping factors ($\lambda$) and volume-dampened aggregation formulas were calibrated to achieve **0.0% FPR** on benign frameworks while maintaining a **0.86 Risk Score** on these mutated payloads.
