Auditoría de Repositorios y Pull Requests

Sentinel se especializa en asegurar tus repositorios conectándose directamente a tu flujo de trabajo. Al conectar un repositorio, Sentinel está al pendiente y examina las propuestas (Pull Requests) que envían otros desarrolladores.

Este escaneo vital ocurre ANTES de que el dueño del repositorio las visualice, las descargue a su máquina local o llegue a fusionarlas a producción, previniendo eficazmente que código malicioso se infiltre en componentes críticos de la aplicación.

sentinel-pr-monitor.exe

Monitoreando repositorio: @usuario/core-app

Nueva PR detectada (#42) de fork externo.

Iniciando pre-análisis de 12 archivos modificados...

PELIGRO: Hook malicioso detectado en dependencias.

Acción: Bloquear Merge / Ocultar Localmente.

Análisis Inicial y Escaneos Configurables

Al añadir un repositorio a Sentinel por primera vez, el sistema realiza un análisis inicial profundo para verificar si ya existía alguna vulnerabilidad o amenaza inyectada, reportándolo de manera inmediata.

Los intervalos de tiempo y la frecuencia con la que se ejecutan los escaneos de repositorios son completamente configurables. Además, cualquier revisión profunda y auditoría a repositorios puede ser ejecutada de manera "on-demand" en el instante que el usuario lo desee.

audit-config.json

{

"initial_scan": true,

"deep_audit_enabled": true,

"scan_interval_hours": 24,

"on_demand_allowed": true

}


> guardado exitosamente.

Bloqueo Local de Instalaciones Peligrosas

En caso de que se omita o se pase por alto una alerta de seguridad generada por el código, y el usuario intente descargar los archivos o ejecutarlos, Sentinel detiene de forma proactiva cualquier instalación de paquetes en la máquina (como dependencias maliciosas de npm).

Funciona como una barrera impenetrable de última línea, cancelando el proceso y protegiendo el entorno de desarrollo local contra la ejecución y propagación de troyanos o scripts ofuscados.

npm install --force

> fetching packages...

[SENTINEL INTERCEPT]

Analizando binarios entrantes...

FIRMA MALICIOSA ENCONTRADA EN: react-dom-parser (typosquatting)

PROCESO MATADO (SIGKILL)

Instalación local cancelada para proteger el entorno.

Protección de Subida y Cuarentena

Durante el envío de aportaciones hacia los repositorios remotos, Sentinel inspecciona exhaustivamente los paquetes y las subidas.

Si durante la subida (push) la herramienta detecta que el paquete va infectado o contiene información sensible expuesta (como llaves de acceso, tokens, o APIs privadas), Sentinel bloquea inmediatamente la subida y manda los archivos específicos a una zona de cuarentena segura. El código inseguro nunca abandonará tu máquina.

git push origin main

> Enumerating objects: 15, done.

> Activando Pre-Push Hook de Sentinel...

Escaneando archivos por secretos...

SECRETO EXPUESTO: AWS_ACCESS_KEY encontrada en src/config.js

Aislando src/config.js -> ./quarantine_zone/

PUSH RECHAZADO.
